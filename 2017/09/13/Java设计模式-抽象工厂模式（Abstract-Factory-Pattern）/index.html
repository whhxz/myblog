Java设计模式-抽象工厂模式（Abstract Factory Pattern）

发表于 2017-09-13 更新于 2025-03-04 分类于 设计模式

抽象工厂概念

抽象工厂相对工厂方法，可以创建一个产品族，而不是单一的产品。也就是在工厂方法上添加了创建其他产品方法。

如图：

  • SeasonFactory：季节工厂
  • SpringFactory：春天工厂
  • SummerFactory：夏天工厂
  • Clothes：衣服
  • Coat:上衣
  • Trousers：裤子
    Java代码如下： 在范例中，如果要创建夏天的一套衣服，直接使用SummerFactory创建上衣和裤子就可以，这样的话身上一套就是夏天的装扮，避免因为出现混搭而出现问题。在实际生活中，电脑主板和CPU的针脚是需要对应，可以使用该抽象方法，避免组别和CUP针脚对不上。

抽象工厂模式优点

  1. 一个工厂车间一系列相关相互依赖的对象，避免出现混合。
  2. 新增产品族时，无需修改系统，比如冬天的衣服。符合“开闭原则”。

抽象工厂模式缺点

  1. 新增产品的时候比较麻烦，比如新增鞋子。

总结

抽象工厂是在工厂方法模式上的增强，一个工厂类可以创建多个产品。
